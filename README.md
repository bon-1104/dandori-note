# 段取りノート

予定と、それに対する準備の期限をひも付けて保存・管理するための一人用アプリです。  
画像とPDFを予定ごとに添付でき、期限前の通知と iPhone 向けの `.ics` 書き出しにも対応しています。

## できること

- 予定を登録する
- 予定ごとに複数の準備期限を手入力で追加する
- jpg / png / pdf を予定に添付する
- 画像はサムネイル、PDFは小さいプレビュー付きカードで確認する
- ブラウザ通知を使って、準備期限の前に知らせる
- `.ics` を書き出して iPhone カレンダーに取り込み、端末通知につなげる
- 予定と添付をまとめてバックアップ書き出し / 復元する

## ファイル構成

- `index.html`: 画面本体
- `styles.css`: UIデザイン
- `app.js`: データ保存、添付、通知、`.ics` 書き出し
- `sw.js`: PWA用のキャッシュ
- `manifest.webmanifest`: ホーム画面追加用の設定
- `serve.py`: ローカル確認用の簡易サーバー

## まず使う手順

1. このフォルダで `python serve.py` を実行します。
2. 表示されたURLをPCブラウザで開きます。
3. iPhoneで使う場合は、同じWi-Fi上でPCのIPアドレス付きURLをSafariで開きます。
4. Safariの共有メニューから「ホーム画面に追加」を選びます。

同じWi-Fiが使えない場合は、Netlify Drop などで一度公開URLに置き、iPhoneで開いてホーム画面に追加してください。

## オフライン利用

- 最初の一度だけネット経由で開く必要があります。
- 一度開いてホーム画面に追加すれば、その後はオフラインでも基本操作を続けられます。
- 保存先は端末内なので、iPhone側で使い始めたら iPhone 上のデータが本体になります。
- Safariのデータ削除や機種変更に備えて、定期的にバックアップ保存してください。

## 通知について

- ブラウザ通知:
  アプリを開いている間、または開き直したタイミングで通知対象を検出して知らせます。
- iPhoneでより確実に通知したい場合:
  各準備または予定の `.ics` ボタンからカレンダー用ファイルを書き出し、iPhoneカレンダーに追加してください。

Webアプリだけでは、iPhoneで完全なバックグラウンド通知を安定して行うのに制約があります。  
そのため、今回はすぐ使える形として「ブラウザ通知 + `.ics` 書き出し」を両方入れています。

## データ保存について

- 予定情報はブラウザの `localStorage` に保存されます。
- 添付ファイルはブラウザの `IndexedDB` に保存されます。
- 同じブラウザ・同じ端末の中で管理される想定です。
- バックアップJSONには予定と添付ファイルの両方が入ります。
